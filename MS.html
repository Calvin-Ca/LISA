LLM 规划器(大脑) LISA ⭐(唯一需微调) 现成工具(RAG / VLM / DB / 模板) 输入 / 输出
⭐ = 唯一需要微调的组件;其余全用现成能力拼装。这张图的作用:让面试官一眼看到 —— LISA 是被规划器编排的一个工具,不是系统本身。直接堵住"这不就是个微调模型吗"的追问。

图 2 · 一次巡检的时序图(工具调用顺序)

用 loop(逐检查项)+ alt(有隐患才验证)体现 Agent 的条件分支与多步循环

sequenceDiagram
    participant U as 巡检输入
    participant P as LLM 规划器
    participant R as 规范库 RAG
    participant L as LISA(微调)⭐
    participant V as VLM 验证
    participant H as 历史台账
    participant W as 工单系统

    U->>P: 工地照片 + 元数据
    P->>R: retrieve_regulation("脚手架区域")
    R-->>P: 检查项[临边防护, 安全帽, 脚手板...]

    loop 每个检查项
        P->>L: segment_hazard(img, "圈出未防护的临边")
        L-->>P: 掩码 + 坐标(候选隐患)
        alt 有候选隐患
            P->>V: verify_region(img, 掩码)
            V-->>P: 确认为真隐患
            P->>H: query_history(位置, 类型)
            H-->>P: 无历史 / 重复 / 已整改
        end
    end

    P->>W: create_work_order(隐患列表)
    W-->>U: 整改工单 + 更新台账

    Note over U,W: 下一周期送入整改后照片 → 复查同位置 → 闭环
时序图的价值:展示这是"调用 → 看结果 → 再决策"的多步循环,而非线性流水线 —— 这正是"为什么必须做成 Agent"的直观证据。
